Feature Engineering

TF-IDF Vectorization & N-Grams

Text to Numbers

Methodology

Term Frequency-Inverse Document Frequency (TF-IDF) converts text into numerical vectors reflecting word importance relative to the entire corpus.

Configuration

  • Unigrams (1-word)
  • Bigrams (2-word sequences)
  • Max Features Limit

Key Advantages

Sparse Data Handling

Ideally suited for SVM & high-dimensional spaces

Context Preservation

Bigrams capture negation (e.g., "not happy")

Noise Reduction

Downweights common words, boosts rare signals

Context Capture Mechanism

Bigram Power

Raw Input

"The movie was not good at all"

Unigrams Only (Lost Context)

movie not good

"good" seen as positive signal

Unigrams + Bigrams

movie not good

Captures negative sentiment

Numerical Output Vector

movie 0.12
not 0.05
good 0.08
not good 0.45

TF-IDF(t, d) = tf(t, d) log(N / df(t))

Model Development

Algorithm Selection & Training Strategy

Data Split Strategy 80% Train 20% Test

Logistic Regression

Baseline Linear Classifier

  • Fast training and inference speed ideal for large datasets
  • Highly interpretable coefficients for feature importance
  • Effective baseline for binary classification tasks

Naive Bayes (Bernoulli)

Probabilistic Model

  • Specifically optimized for text data classification
  • Assumes feature independence (works well with bag-of-words)
  • Bernoulli variant handles binary feature presence efficiently
Top Performer

Linear SVM

Maximum Margin Classifier

  • Maximizes margin between positive and negative classes
  • Superior performance on high-dimensional sparse data
  • Strong generalization capability reducing overfitting
Objective: Maximize accuracy, F1-score, and AUC while minimizing misclassification errors across both sentiment classes.
